# 身份隔离注意事项

> 对应 [mainland-topology.md](./mainland-topology.md) 中「低 KYC 境外 VPS」选型场景。目标：**减少把真实身份、真实住址 IP、日常账号绑在一起**；不是「绝对匿名」，也不是「无法追溯」。

## 边界说明

低 KYC 只表示开户时身份核验较弱，**不等于**服务商不记录、不配合、查不到。Akile 等 VPS 商通常仍会保留：

- 注册邮箱、付款记录、登录 IP
- VPS IP、实例生命周期、工单与滥用投诉记录
- 流量/端口异常等运维日志

在符合法律法规或有效法律程序时，服务商协议通常保留披露账户信息的权利。本文档仅描述**运营侧减少不必要关联**的操作纪律，不构成法律建议。

---

## 核心清单（最小可行）


| #   | 事项                                  | 要点                                              |
| --- | ----------------------------------- | ----------------------------------------------- |
| 1   | **专用邮箱**                            | 不与日常 QQ / 微信 / 主 Gmail 混用；注册、找回、通知均在同一隔离策略下完成   |
| 2   | **专用浏览器配置**                         | 独立 Profile / Container；不登录 Google 主账号、不同步主浏览器数据 |
| 3   | **TUN 全局 VPN + Kill Switch + 泄漏测试** | 系统层全局路由；断线禁止直连；定期测 DNS / WebRTC 泄漏              |
| 4   | **所有 Akile 操作只在 VPN 下**             | 注册、登录、控制台、工单、账单、邮件验证链接点击                        |
| 5   | **支付选与主身份隔离最强的方式并固定**               | 全程同一策略；默认认为支付链路比登录 IP 关联更强                      |
| 6   | **启用 Akile 2FA**                    | 降低账户被盗导致的额外暴露                                   |
| 7   | **工单只说技术问题**                        | 不带真实姓名、住址、主域名或可识别个人/业务身份的信息                     |

## 分项说明

### 1. 专用邮箱 + 专用浏览器配置

- 邮箱 display name 不用真实姓名；尽量不与主手机号绑定。
- 浏览器专用配置仅用于 Akile + 该专用邮箱，避免与日常浏览会话混用。
- 可选：独立浏览器（如 LibreWolf）或独立系统用户，进一步隔离 Cookie / 指纹。

### 2. TUN 全局 VPN + Kill Switch + 泄漏测试

- **TUN 全局**：系统层路由，不要只靠浏览器插件。
- **Kill Switch**：VPN 断线时禁止任何直连，避免真实 IP 一次性写入。
- **固定出口**：Akile 相关操作尽量使用同一 VPN 出口，减少异常登录标记。
- **泄漏测试**（换 VPN 或换设备后必做）：
  - DNS：[dnsleaktest.com](https://dnsleaktest.com)
  - WebRTC：关闭或限制 WebRTC
  - 登录前确认出口 IP 与预期一致

效果：Akile 看到的是 **VPN 出口 IP**，而非家庭/办公宽带 IP；但 VPN IP 本身仍会被记录。

### 3. 所有 Akile 操作只在 VPN 下

适用场景：

- 注册、登录、控制台、提工单、查看账单
- 点击邮件中的验证 / 重置链接
- 手机端操作（若用手机，需同样走全局 VPN，勿用蜂窝网络直连）

**纪律：** 一次未开 VPN 的直连登录，真实 IP 可能永久写入，后续 VPN 无法撤销已记录的数据。

### 4. 支付：隔离最强的方式并固定

按关联强度粗排（强 → 弱）：


| 方式                 | 关联特点                 |
| ------------------ | -------------------- |
| 支付宝 / 信用卡 / Stripe | 强实名，关联最强             |
| USDT 等 crypto      | 链上可追溯，但与日常实名账户分离度相对高 |


- 选定一种方式后**不要混用**。
- 若用 crypto：专用钱包 + 专用地址，避免从 KYC 交易所直接提币到该地址。
- 订单/发票截图存加密离线位置，勿与主云盘混放。



### 5. 启用 Akile 2FA

- Akile 控制台会引导开启 2FA，建议启用。
- 妥善保存恢复码；2FA 与专用邮箱凭证分开备份。

### 6. 工单只说技术问题

- 只写故障现象、节点、套餐、错误信息等技术必要内容。
- 不写：真实姓名、住址、主项目域名、可识别个人身份的业务细节。
- 截图前检查是否泄露桌面其他窗口、邮箱、本地路径。

---



## 验收标准

**Akile 侧应看到：**

- 一个 VPN 出口 IP
- 专用邮箱
- 独立且固定的支付链路
- 技术向工单内容

**不应出现：**

- 家庭/办公宽带直连 IP
- 主邮箱、主支付宝、主手机号
- 可识别个人身份的工单或截图信息

---



## 明确不要做的事

- 不要把「全局 VPN」理解成「Akile 无记录」—— 出口 IP 仍会被记录。
- 不要用主邮箱 + 支付宝 + 家庭宽带直连的组合。
- 不要在 Telegram / 社交账号上用主号讨论 Akile 账户细节。
- 不要把 Akile VPS IP 直接绑到可识别个人的域名或商户信息。
