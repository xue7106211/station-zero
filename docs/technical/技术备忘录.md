存储的关键：
- 关键不是「存在哪」，而是「用户浏览器请求的 URL 是什么」
- 用户打开 /movies/dune-part-two，页面里可能有：<img src="???">
- 浏览器只会对 ??? 发 HTTPS 请求
- 「存在哪」是运维/架构的事；「请求的 URL 是什么」才是大陆用户体感的事
- URL A：直连 Supabase（差）
  - 用户浏览器 ──→ *.supabase.co（Supabase 公网端点）
  - 还暴露「你们用 Supabase」、bucket 结构
- URL B：本站相对路径 + Next 静态目录（开发期 OK）
  - <img src="/media/posters/dune-part-two.jpg">
  - 域名是 你们自己的站
  - 大陆体验取决于 整站部署在哪、有没有 CDN
- URL C：媒体子域 + CDN（生产目标）
  - <img src="https://media.station-zero.com/posters/dune-part-two.webp">
  - 用户 从未请求 supabase.co
  - 大陆快慢主要看 CDN 供应商和 PoP，不是 Storage 品牌
  - 文件仍在 Supabase Storage，但 URL 是 media 子域，体验由 CDN 决定
