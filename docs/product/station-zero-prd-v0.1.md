# Station Zero 零号站产品需求文档 v0.1

> 文档状态：讨论稿  
> 当前阶段：产品定义与需求沉淀  
> 重要边界：本文档不代表进入工程初始化或开发实施阶段。

## 1. 产品定位

**Station Zero 零号站** 是一个面向高清观影用户的观影决策工具。

它帮助用户回答三个问题：

1. **这部片值不值得看？**
2. **现在可以在哪里合法观看？**
3. **哪个高清版本最值得看？**

一句话定位：

> Station Zero 零号站，是一个帮用户判断「一部片值不值得看、哪里能合法看、哪个高清版本最值得看」的高清观影决策系统。

## 2. 产品背景

现有影视产品大致分散在几个方向：

- 豆瓣、时光网、WMDB 解决影视资料、评分、评论、片单问题。
- 猫眼、淘票票解决院线购票和上映信息问题。
- 各类影视资源站解决资源查找问题，但合规风险高、体验粗糙、信任感弱。
- Jellyfin、Plex、Emby、TinyMediaManager 解决私人片库管理问题。
- 公众号、B 站、小红书、知乎影视内容解决推荐和种草问题。

但目前缺少一个产品，把以下能力整合起来：

- 影视发现
- 正版观看路径
- 高清版本判断
- 高质量片单策展
- 设备与场景化观影建议
- 后续私人片库管理

## 3. 目标用户

### 3.1 核心用户

第一阶段优先服务：**高清观影玩家**。

他们通常具备以下特征：

- 关注 4K、HDR、杜比视界、蓝光、REMUX、WEB-DL 等版本差异。
- 可能拥有电视、投影、回音壁、家庭影院、NAS 或私人片库。
- 不满足于“能看就行”，而是关心“用什么方式看最值得”。
- 愿意为节省判断时间、获得更好观影体验付费。

### 3.2 次级用户

后续可以扩展到：

- 普通影迷：想知道一部片值不值得看、在哪里能看。
- 电影审美用户：喜欢高质量片单、导演专题、类型推荐。
- NAS 用户：需要管理自己的本地片库和高清版本信息。

## 4. 核心用户问题

用户在决定观看一部影片时，常见痛点包括：

1. **信息分散**：评分在豆瓣，观看平台在另一个地方，高清版本信息又在其他论坛或资料页。
2. **版本难判断**：用户不清楚 4K、HDR10、Dolby Vision、BluRay、WEB-DL、REMUX 的差异，也不知道某部片是否值得追求高规格版本。
3. **正版路径不清晰**：用户不知道国内外哪些平台可以合法观看，也不知道当前是订阅、租赁、购买、院线、影展还是实体碟发行。
4. **传统资源站体验粗糙**：页面常堆砌链接和广告，信任感弱，信息可能过期、错误或存在安全风险。
5. **观影建议不够场景化**：用户不只是想知道评分，也想知道适合什么设备、心情、时间和观看环境。

## 5. 产品目标

### 5.1 第一阶段目标

构建一个可浏览的高清影视指南网站，重点验证：

- 单部影片详情页是否能形成差异化价值。
- 用户是否认可「高清版本判断 + 正版观看路径 + 策展推荐」的组合。
- Station Zero 是否能从资源站、评分社区、影视媒体中建立独立定位。

### 5.2 第一阶段不做什么

第一阶段不做：

- 不做完整社区。
- 不做积分系统。
- 不做用户投稿审核系统。
- 不做完整账号和会员系统。
- 不做自动化大规模数据采集。
- 不做复杂 NAS 管理工具。

## 6. MVP 范围

第一版 MVP 聚焦四类页面。

### 6.1 首页

首页承担品牌解释和内容入口功能。

核心模块：

- 产品主张：高清观影决策系统。
- 精选影片：展示若干高质量影片决策卡片。
- 高清版本追踪：展示近期值得关注的 4K / HDR / 蓝光版本。
- 策展片单：围绕导演、类型、设备、心情、场景组织内容。
- 高清知识入口：解释常见高清格式和观看概念。

首页目标：

- 让用户感受到专业、审美和可信。
- 引导用户进入影片详情页。

### 6.2 影片详情页

影片详情页是 MVP 的核心。

它不只是资料页，而是一个观影决策界面。

#### A. 观影决策摘要

第一屏优先回答：

- 是否推荐观看？
- 最适合什么方式观看？
- 哪里可以合法观看？
- 有没有值得追求的高清版本？
- 适合什么场景？
- 不适合什么用户？

示例：

> 推荐指数：值得大屏观看  
> 最佳观看方式：4K Dolby Vision + 好声音系统  
> 合法观看路径：Apple TV / Prime Video / 蓝光发行  
> 适合场景：周末夜晚、家庭影院、科幻视觉体验  
> 不适合：只想轻松下饭、对慢节奏不耐烦

#### B. 基础资料

包含：

- 片名
- 海报
- 年份
- 国家/地区
- 类型
- 导演
- 主演
- 片长
- 简介
- 评分信息

#### C. 正版观看路径

展示：

- 流媒体平台
- 租赁/购买平台
- 院线/影展状态
- 蓝光/4K UHD 发行信息
- 官方发行方或厂牌信息

#### D. 高清版本判断

展示该片的高清版本情况，例如：

- 是否有 4K
- 是否有 HDR10
- 是否有 Dolby Vision
- 是否有 IMAX 画幅
- 是否有 BluRay
- 是否有 4K UHD Blu-ray
- 是否有修复版
- 是否值得追求高规格版本

重点不是堆参数，而是给判断：

> 这部片的摄影和视效值得优先选择 4K HDR 版本。  
> 如果只是移动设备观看，普通高清流媒体版本已经足够。  
> 如果使用投影或 OLED 电视，建议优先选择杜比视界或蓝光版本。

#### E. 设备与场景建议

根据影片类型和版本特点，给出观看建议：

- 手机可看
- 电视推荐
- 投影推荐
- 耳机推荐
- 音响系统推荐
- 家庭影院强推荐
- 适合独自观看
- 适合情侣观看
- 适合周末夜晚
- 适合下饭
- 不适合碎片时间

#### F. 相关推荐

推荐逻辑包括：

- 同导演
- 同演员
- 同类型
- 同视觉风格
- 同主题
- 同观看场景
- 同高清体验价值

### 6.3 片单页

片单不是简单合集，而是策展内容。

片单方向可以包括：

- 适合 OLED 电视看的电影
- 值得用投影看的科幻片
- 杜比视界体验片单
- 周末夜晚沉浸式观影
- 不用动脑的高完成度商业片
- 适合测试音响系统的电影
- 被低估的 4K 修复老片
- 同一个导演的最佳入门顺序

片单目标：

- 建立 Station Zero 的审美判断。
- 提供比“排行榜”更有场景感的发现体验。
- 为后续会员高级筛选做铺垫。

### 6.4 高清知识库

知识库用于降低用户理解成本。

首批可以解释：

- 4K 是什么
- HDR 是什么
- HDR10 和 Dolby Vision 有什么区别
- BluRay 和 WEB-DL 有什么区别
- REMUX 是什么
- IMAX 画幅是什么
- 为什么同一部片会有多个版本
- 什么时候值得追求蓝光或 4K UHD
- 什么设备才能看出 HDR 的价值

知识库目标：

- 建立专业信任。
- 降低新用户门槛。
- 支撑影片页中的版本判断。

## 7. 核心信息架构

建议首版导航结构：

- 首页
- 影片
- 片单
- 高清知识
- 版本追踪
- 关于零号站

其中：

- **影片** 是核心资料和决策入口。
- **片单** 是内容策展入口。
- **高清知识** 是专业信任入口。
- **版本追踪** 是未来会员和订阅提醒的前置入口。
- **关于零号站** 用来明确合规边界和产品理念。

## 8. 内容数据策略

第一阶段采用 **半人工策展**。

原因：

- 可以保证信息质量。
- 可以快速验证产品体验。
- 不需要一开始处理复杂数据源、版权、反爬、审核和自动更新问题。
- 更符合产品早期“建立判断力”的目标。

首批内容建议：

- 20–50 部高质量影片。
- 5–10 个策展片单。
- 10–20 篇高清知识条目。
- 若干近期值得关注的 4K / HDR / 蓝光发行信息。

优先选择：

- 视觉和声音体验强的电影。
- 已有清晰正版观看路径的电影。
- 有典型版本差异的电影。
- 适合解释高清概念的电影。

## 9. 视觉与体验方向

视觉气质：**杂志策展感**。

关键词：

- 电影感
- 克制
- 高级
- 有判断力
- 信息清晰
- 不像资源站
- 不像普通数据库
- 不像纯工具后台

体验原则：

- 第一屏给结论，不让用户自己在信息里挖答案。
- 参数服务于判断，不为了炫技堆规格。
- 页面要有编辑判断，而不是机械展示。
- 影片页要像“观影顾问”，不是“资源详情页”。
- 片单页要像“策展专题”，不是“分类列表”。

## 10. 商业化方向

会员制不卖资源链接。

可会员化能力包括：

- 无广告体验
- 高级筛选
- 收藏影片
- 私人片单
- 影片订阅提醒
- 剧集更新提醒
- 4K / HDR / 蓝光版本追踪
- 私人片库管理
- NAS 元数据整理
- 设备观影方案
- 高清知识进阶内容

用户付费购买的是：

- 节省时间
- 降低判断成本
- 更好的观影体验
- 更可信的高清版本信息
- 更结构化的私人片库管理

## 11. 成功指标

第一阶段可以观察：

### 11.1 内容指标

- 用户是否愿意点进影片详情页。
- 用户是否阅读高清版本说明。
- 用户是否从影片页跳转到正版观看路径。
- 用户是否收藏或分享片单。

### 11.2 产品指标

- 首页到影片页点击率。
- 影片页停留时长。
- 片单页点击率。
- 高清知识页阅读完成度。
- 搜索或筛选使用率。

### 11.3 定性指标

- 用户是否能准确复述产品价值。
- 用户是否认为它比资源站更可信。
- 用户是否认为它比豆瓣更适合高清观影决策。
- 用户是否愿意为版本追踪或私人片库管理付费。

## 12. 后续路线

### Phase 1：高清观影指南

重点：

- 首页
- 影片详情页
- 片单
- 高清知识库
- 半人工内容维护

目标：验证产品定位和内容体验。

### Phase 2：版本追踪与订阅

重点：

- 影片关注
- 剧集更新提醒
- 4K / HDR / 蓝光发行提醒
- 新上线平台提醒

目标：从内容站升级为工具型产品。

### Phase 3：私人片库工具

重点：

- 用户已有影片记录
- 本地片库导入
- NAS 元数据整理
- 版本识别
- 字幕、海报、元数据管理建议

目标：进入高价值用户场景。

### Phase 4：会员体系

重点：

- 高级筛选
- 私人片单
- 订阅提醒
- 片库管理
- 无广告
- 专业知识内容

目标：建立可持续商业模式。

## 13. 合规边界

Station Zero 零号站第一版必须保持清晰合规边界：

- 公开产品只展示正版观看路径、官方发行信息、合法资料来源和高清版本知识。
- 不提供磁力、BT、网盘、迅雷、盗版资源站或侵权下载入口。
- 不以下载资源链接作为会员权益或核心卖点。
- 不把灰色资源站外链作为流量增长策略。
- 不鼓励用户绕过版权方、平台或发行方获取内容。

产品长期信任感来自专业判断、信息组织、审美策展和合法观看导航，而不是资源链接数量。

## 14. 当前版本结论

Station Zero 零号站第一版不应该追求“大而全”的影视数据库，也不应该复制资源站。

它应该先把一个页面做好：

> 当用户打开一部影片，Station Zero 能比任何资源站、评分网站或普通影视资料页更快告诉他：  
> 这部片值不值得看，应该在哪里合法看，哪个高清版本最值得看。

这就是 MVP 的核心。
