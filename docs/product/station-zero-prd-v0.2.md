# Station Zero 零号站产品需求文档 v0.2

> **相对 v0.1 的核心变更**：明确本站提供**在线聚合式影视资料与资源索引**（含网盘、迅雷、磁力等外链），但**不提供视频存储与托管**；用户在本站查阅信息后，自行跳转至第三方链接下载或观看。  
> 当前阶段：产品定义与需求沉淀。  
> 重要边界：本文档指导产品方向，具体实现以仓库代码与 `AGENTS.md` 为准。

## 0. 版本变更摘要

| 维度 | v0.1 | v0.2 |
|------|------|------|
| 资源链接 | 明确不提供网盘 / 磁力 / 迅雷 | **提供聚合索引**，按类型分组展示外链 |
| 服务性质 | 偏「正版观看导航 + 高清决策」 | **影视信息整理与索引服务** + 高清决策 |
| 视频托管 | 未强调 | **明确不做**：不存储、不上传、不制作视频文件 |
| 合规表述 | 禁止展示灰色链接 | 索引服务定位 + 侵权投诉与下架机制 |

## 1. 产品定位

**Station Zero 零号站** 是一个面向高清观影用户的**影视信息整理、资源索引与观影决策**服务。

它帮助用户回答四个问题：

1. **这部片值不值得看？**
2. **哪个高清版本最值得看？**
3. **现在可以在哪里合法观看？**（流媒体、租赁、购买、实体发行等）
4. **哪里能找到整理好的下载资源？**（网盘、迅雷、磁力等外链索引）

一句话定位：

> Station Zero 零号站，是一个帮用户判断「一部片值不值得看、哪个版本值得追、哪里能合法看、哪里能找到整理好的资源链接」的高清影视索引与观影决策系统。

### 1.1 我们是什么 / 不是什么

| 我们是 | 我们不是 |
|--------|----------|
| 影视资料与资源链接的**聚合索引** | 视频托管、网盘、BT 站或下载服务 |
| 帮用户节省查找与判断时间的**决策界面** | 存储、分发或制作任何视频文件的一方 |
| 按平台 / 类型整理外链的**信息整理服务** | 对用户下载行为负责的平台 |

用户在本站完成「查找 → 判断 → 复制或跳转链接」，**实际下载与观看发生在第三方**，由用户自行承担相应责任。

## 2. 产品背景

现有影视产品大致分散在几个方向：

- 豆瓣、时光网、WMDB：资料、评分、评论、片单。
- 猫眼、淘票票：院线与购票。
- 各类影视资源站：资源查找，但页面粗糙、广告多、版本信息混乱、信任感弱。
- Jellyfin、Plex、Emby：私人片库管理。
- 公众号、B 站、小红书：推荐与种草。

用户仍缺少一个产品，把以下能力**在同一部影片详情页**整合起来：

- 影视发现与策展推荐
- 高清版本判断与设备建议
- 正版观看路径（流媒体 / 购买 / 实体）
- **网盘、迅雷、磁力等资源的聚合索引**（外链跳转，本站不托管）
- 高质量片单与高清知识

v0.2 将「资源索引」从竞品式粗糙列表，升级为**有判断、有分类、有版本语境**的决策体验——这是与传统资源站差异化的关键。

## 3. 目标用户

### 3.1 核心用户

第一阶段优先服务：**高清观影玩家**。

特征：

- 关注 4K、HDR、杜比视界、蓝光、REMUX、WEB-DL 等版本差异。
- 拥有或计划使用电视、投影、NAS、私人片库。
- 需要在网盘、磁力、迅雷等多渠道间快速找到**可信、带版本说明**的链接。
- 不满足于资源站「只有链接没有判断」，愿意为节省筛选时间付费。

### 3.2 次级用户

- 普通影迷：想知道值不值得看、去哪看、去哪下。
- 策展向用户：高质量片单、导演专题、类型推荐。
- NAS 用户：需要版本信息与元数据整理（后续阶段）。

## 4. 核心用户问题

1. **信息分散**：评分、正版平台、网盘分享、磁力种分散在论坛、群组、多个资源站。
2. **版本难判断**：同一部片有 1080p / 4K / HDR / 不同压制组，用户不知道哪个值得下。
3. **资源站体验差**：链接堆砌、失效多、广告干扰、缺少影片层面的观影建议。
4. **正版路径仍不清晰**：部分用户同时需要合法流媒体选项与本地高清收藏路径。
5. **信任与合规焦虑**：用户希望站点立场清晰——是索引而非盗版托管。

## 5. 产品目标

### 5.1 第一阶段目标

构建可浏览的高清影视索引站，重点验证：

- 详情页能否同时承载「观影决策 + 资源索引」而不沦为粗糙资源站。
- 用户是否认可「版本判断 + 正版路径 + 聚合外链」的组合。
- 聚合式资料（含网盘 / 迅雷 / 磁力）能否成为**核心留存能力**之一。

### 5.2 第一阶段不做什么

- 不提供视频文件存储、转码、在线播放或离线下载加速。
- 不做用户上传视频或本站托管种子文件。
- 不做完整社区、积分、完整账号体系（可后续迭代）。
- 不做自动化大规模侵权采集；批量录入需配合审核与下架机制（见 §13）。
- 不做复杂 NAS 管理工具（后续阶段）。

## 6. MVP 范围

### 6.1 首页

与 v0.1 一致，承担品牌解释与内容入口：

- 产品主张：高清影视索引与观影决策。
- 精选影片、策展片单、高清知识入口。
- 明确传达：**整理与索引服务，非视频托管**。

### 6.2 影片详情页（核心）

详情页是 MVP 核心，结构为「决策摘要 + 资料 + 观看来源聚合」。

#### A. 观影决策摘要

第一屏优先回答：

- 是否推荐观看？
- 最适合什么方式 / 规格观看？
- 有没有值得追求的高清版本？
- 适合什么场景？不适合谁？

#### B. 基础资料

片名、海报、年份、类型、导演、主演、片长、简介、评分等（TMDB + 人工策展）。

#### C. 观看来源（聚合模块）

按**固定分类顺序**展示外链，用户点击后跳转第三方或复制链接：

| 分类 | 说明 | 数据来源（典型） |
|------|------|------------------|
| 订阅 | 流媒体订阅观看 | TMDB watch providers / 人工补全 |
| 租赁/购买 | 数字租赁或购买 | 同上 |
| 实体发行 | 蓝光 / 4K UHD 等 | 人工 / 发行信息 |
| 网盘 | 夸克、百度、阿里等分享链接 | **人工整理** |
| 迅雷 | 迅雷云盘 / 迅雷下载链接 | **人工整理** |
| 磁力 | magnet 链接 | **人工整理** |
| 资料来源 | IMDb、豆瓣等参考页 | 人工 |

**交互原则：**

- 每条链接展示：平台名、类型标签、备注（体积、分辨率、字幕、压制组等）。
- 磁力链可复制；网盘 / 迅雷跳转至第三方页面。
- 模块底部展示免责声明摘要，链至关于页完整表述。
- 失效链接应可标记、下架或更新（运营流程，首版可人工维护）。

#### D. 高清版本判断

与 v0.1 相同：不只堆参数，给出「是否值得追 4K / HDR / 蓝光」的判断。

#### E. 设备与场景建议

与 v0.1 相同。

#### F. 相关推荐

与 v0.1 相同。

### 6.3 片单页

策展片单，强调场景与审美判断。

### 6.4 高清知识库

解释 4K、HDR、BluRay、REMUX 等概念，支撑版本判断与资源选择。

### 6.5 关于页（合规必备）

必须包含完整服务声明与侵权投诉通道（见 §13.2），并在全站页脚提供入口。

## 7. 核心信息架构

首版导航：

- 首页
- 影片
- 片单
- 高清知识
- 关于零号站

「版本追踪」等能力保留为后续 Phase，不在 v0.2 MVP 强制范围。

## 8. 内容数据策略

### 8.1 策展 + 批量索引

- **影片元数据与决策层**（`verdict`、`bestWay` 等）：半人工策展，保证判断质量。
- **正版观看路径**：TMDB 同步 + 人工补全。
- **网盘 / 迅雷 / 磁力**：人工整理或批量导入后**审核发布**；staging 数据与前台展示分离（见技术方案 `bulk-ingestion-scheme`）。
- 首版可先覆盖 20–50 部精品 + 规模化录入流水线验证。

### 8.2 资源条目字段（建议）

与 `ViewingPath` 模型对齐：

```ts
type ViewingPath = {
  platform: string;   // 如「夸克网盘」「迅雷」「磁力」
  type: "订阅" | "租赁/购买" | "实体发行" | "网盘" | "迅雷" | "磁力" | "资料来源";
  url?: string;       // 外链；磁力为 magnet:? 开头
  note?: string;      // 体积、分辨率、字幕、版本说明
};
```

**v0.2 新增类型说明：** `迅雷` 作为独立分类，与 `网盘`、`磁力` 并列展示；若工程上暂未单独枚举，可先归入 `网盘` 备注，但产品语义上应区分。

### 8.3 展示优先级

详情页「观看来源」推荐顺序：

> 订阅 → 租赁/购买 → 实体发行 → 网盘 → 迅雷 → 磁力 → 资料来源

正版路径优先展示；资源索引紧随其后，避免页面气质完全滑向纯资源站，同时满足核心用户需求。

## 9. 视觉与体验方向

关键词：**杂志策展感 + 清晰索引**。

- 第一屏给**观影判断**，不只是链接列表。
- 资源区分组用克制标签，避免满屏广告式堆砌。
- 不像传统资源站：无弹窗、无误导下载按钮、无本站托管伪装。
- 每条资源行信息密度适中：平台 + 备注 + 单一明确操作（跳转 / 复制）。
- 详情页整体仍是「观影顾问」，资源索引是其中一个结构化模块，而非整页唯一内容。

## 10. 商业化方向

会员制**不卖资源文件**，可售卖：

- 无广告体验
- 高级筛选（分辨率、HDR、字幕、平台类型）
- 收藏与片单
- 版本 / 链接更新提醒
- 私人片库管理（后续）
- 高清知识进阶内容

用户付费购买的是：**判断能力、索引质量、更新及时性与使用体验**，不是网盘空间或下载带宽。

## 11. 成功指标

### 11.1 内容指标

- 详情页「观看来源」模块点击率 / 复制率。
- 网盘 / 磁力 / 迅雷分类的交互占比。
- 资源链接失效后的修复时长（运营）。
- 用户是否仍阅读版本判断与场景建议（避免沦为纯资源站）。

### 11.2 产品指标

- 首页 → 详情页 CTR。
- 详情页停留时长。
- 片单与知识库渗透率。
- 搜索与筛选使用率。

### 11.3 合规指标

- 侵权投诉响应时效。
- 下架 / 断开链接执行记录。
- 关于页与免责声明曝光（页脚可达）。

## 12. 后续路线

### Phase 1：高清索引与决策（当前）

- 详情页聚合模块（正版 + 网盘 / 迅雷 / 磁力）
- 半人工 + 批量录入 staging
- 合规页与投诉通道

### Phase 2：规模与质量

- 万级影片索引
- 链接失效巡检、版本标注规范化
- 订阅提醒（正版上线 / 新资源版本）

### Phase 3：私人片库工具

- 本地 / NAS 元数据整理（后续）

### Phase 4：会员体系

- 高级筛选、收藏、提醒、无广告

## 13. 合规边界与服务声明

### 13.1 服务性质（全站必须传达）

本站为**影视信息整理与索引服务**：

- **不**主动上传、存储或制作任何未经授权的视频文件。
- **不**提供视频托管、在线播放、离线下载或种子分发能力。
- 站内影片资料、海报、文字及部分链接来自 TMDB 同步、人工整理、用户提交或公开网络资料。
- 展示的外链（含网盘、迅雷、磁力）均指向第三方，由用户自行访问与下载；本站仅作信息聚合与跳转。

### 13.2 侵权投诉与处理（关于页 / 页脚原文）

以下表述应在关于页显著展示，并在页脚提供「版权投诉」或「侵权通知」入口：

---

**版权与侵权通知**

本站为影视信息整理与索引服务，不主动上传、存储或制作任何未经授权的视频文件。站内部分信息可能来自用户提交或公开网络资料。

如您认为本站展示的内容、链接、图片、文字或其他信息侵犯了您的合法权益，请通过 **copyright@example.com** 与我们联系，并提供：

- 权属证明
- 侵权链接（本站具体页面 URL）
- 有效联系方式

我们将在收到有效通知后，依法及时核查，并根据情况采取**删除、屏蔽、断开链接**等必要措施。

---

> **运营备注**：上线前将 `copyright@example.com` 替换为项目专用邮箱；该邮箱应与 WHOIS / 运营身份策略一致（见 `mainland-topology` 隐私目标）。

### 13.3 运营原则

- 不将「链接数量」作为唯一增长指标；优先保证元数据质量、版本标注与投诉响应。
- 批量导入内容须经 staging 与抽检，避免明显侵权或恶意链接直达前台。
- 收到有效侵权通知后，优先处理：断开链接 → 下架条目 → 必要时下架整部影片页。
- 会员权益不得包含「独家资源」「高速下载」等暗示本站托管能力的表述。
- 页面文案避免鼓励用户规避版权方合法渠道；正版路径与资源索引并列展示时，保持中立、信息型语气。

### 13.4 与 v0.1 合规章节的关系

v0.1 §13 采取「公开产品不提供灰色链接」的保守策略；**v0.2 废止该条**，改为「索引 + 不托管 + 投诉下架」框架。若法务或运营策略变化，以届时生效的 PRD 版本与关于页公示为准。

## 14. 当前版本结论

Station Zero v0.2 的核心不是再做一张「链接大表」，而是：

> 当用户打开一部影片，Station Zero 能比传统资源站更快告诉他**值不值得看、哪个版本值得追**；比评分站更清楚**去哪合法看、去哪找整理好的网盘 / 迅雷 / 磁力链接**——且这一切发生在**专业、克制、可投诉下架**的索引服务边界内。

**关键方向（v0.2 必达）：**

1. 提供在线、聚合式资料，含网盘、迅雷、磁力等资源外链索引。
2. 不提供视频存储；用户自行跳转第三方下载或观看。
3. 服务声明与侵权投诉机制产品化（关于页 + 页脚）。

这就是 v0.2 的产品基线。
